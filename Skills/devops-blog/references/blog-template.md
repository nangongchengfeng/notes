# DevOps 博客模板

## 故障复盘类模板

```markdown
# [具体问题]故障排查与修复：我们踩过的那些坑

## 引言

[一句话场景代入：时间、地点、发生了什么]

[说明这篇文章会讲什么，读者能学到什么]

## 故障现象

### 时间线
- **[时间1]** - [发生了什么：告警、现象]
- **[时间2]** - [后续发展]
- **[时间3]** - [最严重的时刻]

### 环境信息
- 集群规模：[X 节点 / Y Pod / Z QPS]
- 技术栈：[K8s 1.2x / Java 17 / Redis 7.x]
- 影响范围：[哪些服务 / 多少用户]

### 监控数据
[放一个简单的 Mermaid 图或文字描述趋势]

## 排查过程

### 第一步：[先查了什么]
- 做了什么操作：`命令`
- 看到了什么现象
- 当时的想法和误判

### 第二步：[又查了什么]
- 做了什么操作：`命令`
- 关键发现
- 转折点

### 第三步：[定位根因]
- 最终怎么找到的
- 根因是什么

## 根因分析

[深入分析为什么会发生这个问题]

- 技术层面：[代码/配置/架构问题]
- 流程层面：[为什么没发现 / 为什么没监控到]
- 人为层面：[有什么可以改进的]

## 解决方案

### 临时止血
- [做了什么快速恢复]
- `相关命令或配置`

### 长期根治
- [最终的修复方案]
- `代码/配置示例`
- 为什么选这个方案

## 经验总结

### 1. [第一条经验]
[具体内容]

### 2. [第二条经验]
[具体内容]

### 3. [第三条经验]
[具体内容]

### 附：排查命令清单
```bash
# 命令1
# 命令2
# 命令3
```

## 总结

[简短总结，给出建议]
```
